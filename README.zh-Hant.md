## 這是一個 Vibe Code 專案

這個 repo 屬於 vibe code：一個想到什麼做什麼的專案。它的存在是為了讓臨時的小想法可以活起來，而不是為了把程式寫得多完美。

如果您需要找一個程式碼品質良好的專案，請不要找目前這個。用 vibe 的方式做它，最主要的原因就是：這專案從來不是為了寫得多完美，只是想讓一個一閃而過的小念頭能繼續呼吸。

話雖如此，底下的意圖才是最低底線。程式碼品質可以隨意漂流，但願景與美學不行。

## 願景與核心概念（不可違反的底線）

真正的主題：人生隨波逐流。寧靜不是風格，是結果，是那個靜觀的觀測者，在身不由己的潮汐之中，把深海的重量萃取成的那一抹光。

意象系統是整個專案的主軸，後面所有設計都掛在這條線上：

| 階段                   | 意象               | 對應的東西          |
| ---------------------- | ------------------ | ------------------- |
| 潮汐 / 深海 / 隨波逐流 | 沉、身不由己、漂泊 | 隧道行走 = 載入過程 |
| 萃取 / 煉金            | 把粗重蒸餾成精純   | 走出隧道的瞬間      |
| 一抹平穩的微光         | 那盞不動、穩定的燈 | 目的地：可棲身的光  |
| 歇腳、補充、再出發     | 潮有漲退，光是中途 | 停留，然後回到潮裡  |

關於概念，再多幾點：

- 名稱呼應：Lantern（不動的穩定微光）立於 Tide（不停變動的潮汐）之中。
- 目的地不是終點，是補給站。漂泊到此休息、補充，然後再出發，所以體驗帶有循環節奏，不是「抵達後定居」。
- 以人為本：這是給人使用的地方，不為機器人最佳化。首頁可被索引，其餘內容希望由人類抵達。
- 關鍵洞見：入口的「走過隧道」這個動作，本身就是天然的人類門檻。爬蟲不會走過隧道，所以美學概念與反爬蟲目標其實是同一件事。

## 設計原則

1. 流暢優先。Jank（卡頓）會直接摧毀寧靜感。簡單而流暢勝過華麗而卡頓。真正致命的是偶發的微頓，而不是低 FPS。
2. 以人為本，守住入口。不是「鎖內容」，只是 (a) 不當機器人的資料源、(b) 所有人都從正門（隧道）進場，不從中途空降破壞流程。
3. 空間連續性。穿過隧道一次之後就不再「離開」；在空間內移動都是平滑轉場，不是重載。
4. 靜態優先，後端解耦。全站是純前端（HTML / CSS / JS，可由 TS 編譯），不強制有後端，可純靜態部署。

## 美學底線

意圖：安靜、暖、慢、低對比但清晰。一盞穩定的光，對著不停變動的深。

唯一刻意的險招，把 boldness 全花在這裡：反轉一般「暖奶油底」的直覺，以「深」為地、以「暖光」為圖。深海與夜是世界，圖書館與咖啡廳是深裡被點亮的暖室，而那一盞會呼吸的微光是全站唯一的記憶點與熱點。其餘一切保持安靜克制，像出門前拿掉一個配件。

- 低彩度貫穿全程；深色為地、暖色為被點亮的圖。
- 動態主張：世界緩緩漂、光不動。背景可有極細微的、像在水下的飄移，而那盞微光始終定住。動中之靜就是這個對比。
- 字體偏暖意的 old-style 襯線；強調用斜體或顏色而非粗體，因為粗重與寧靜相衝。
